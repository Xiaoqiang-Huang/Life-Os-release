# Life OS Pro 购买、授权码与账号中心使用说明

本文档说明用户如何购买 Pro、授权码如何交付、如何自动激活、如何找回授权码，以及运营后台如何查询和补发授权。

## 一、用户购买流程

### 1. 打开 Pro 授权中心

在 Obsidian 中打开 Life OS 插件，进入侧边栏的 `Pro 授权` 页面。

用户需要先填写：

- 授权服务地址：`https://license.lifeoskit.com`
- 邮箱：用于订单绑定、授权找回和账号中心登录

填写后点击保存。

### 2. 选择购买档位

当前支持：

| 档位 | 价格 | 时长 | 设备数 | 定位 |
|---|---:|---|---:|---|
| 30 天试用 | 免费一次 | 30 天 | 最多 3 台 | 完整体验 Pro |
| 月付 Pro | 9.9 元 | 30 天 | 最多 3 台 | 短期 Pro 使用 |
| 买断 Pro | 49 元 | 永久 | 最多 5 台 | 长期 Pro 使用 |

月付 Pro 和买断 Pro 点击后会创建支付宝订单。

### 3. 支付

订单创建成功后，插件会自动打开网页支付页面。如果浏览器阻止自动打开，插件页面中会显示支付入口，用户可以手动点击 `打开网页支付`。手机端 Obsidian 如果没有拉起外部浏览器，用户可以复制支付链接，再粘贴到手机浏览器里打开。

如果支付通道只返回二维码，同一台手机可能无法扫描自己的屏幕。插件端需要显示提示：建议用另一台设备扫码，或进入账号中心/联系支持处理。

如果点击支付时出现类似：

```text
支付通道暂时不可用：商户账户余额不足，无法创建支付订单。
```

这不是用户支付宝余额不足，而是支付通道商户侧没有成功创建订单。此时用户无法继续付款，需要开发者处理支付通道配置或账户余额。

### 4. 自动激活

支付完成后，用户回到 Obsidian，插件会自动轮询订单。

正常情况下，用户不需要手动复制授权码。插件会自动完成：

- 获取授权码
- 激活当前设备
- 保存授权状态
- 显示到期时间和设备数

如果自动激活失败，插件会保存已返回的授权码，用户可以稍后在 `激活码 / 授权码` 区域手动激活。

## 二、核心规则

Life OS Pro 的授权不是依赖用户手动复制邮件里的激活码完成的。

当前正式流程是：

1. 用户在插件内填写邮箱并创建订单。
2. 用户通过支付宝完成支付。
3. 插件自动轮询订单状态。
4. 支付成功后，授权服务返回授权码。
5. 插件自动用授权码激活当前设备。
6. 授权码保存在插件本地设置中，用户后续可导出备份或在账号中心找回。

邮箱的作用是绑定订单、绑定授权、登录账号中心和找回授权码。当前系统只通过邮件发送验证码，不会在支付成功后主动把授权码发到邮箱。

## 三、授权码是如何给到用户的

授权码有三种交付方式。

### 1. 插件自动获取并激活

这是主流程。

创建订单接口会返回 `orderClaimToken`。支付成功后，插件会把这个领取令牌放进请求体的 `claimToken` 字段轮询订单：

```http
POST /api/orders/:orderId
Content-Type: application/json

{
  "claimToken": "创建订单时返回的订单领取令牌"
}
```

只有 `claimToken` 匹配，并且订单已经支付成功，授权服务才会返回 `licenseKey`。插件随后自动调用：

```http
POST /api/licenses/activate
```

完成当前设备激活。

这样可以避免用户只凭订单号枚举或领取别人的授权码。用户后续更换设备、重装插件或更新插件时，也可以通过账号中心找回已购买授权。

### 2. 用户账号中心找回

用户可以打开：

[https://license.lifeoskit.com/portal](https://license.lifeoskit.com/portal)

使用购买时填写的邮箱登录。系统会发送邮箱验证码。登录后用户可以查看：

- 授权码
- 授权档位
- 到期时间
- 最大设备数
- 已激活设备列表

账号中心用于找回授权码、换设备和核对授权状态。

正式用户入口是：

- 用户账号中心：[https://license.lifeoskit.com/portal](https://license.lifeoskit.com/portal)
- 运营后台：[https://license.lifeoskit.com/admin](https://license.lifeoskit.com/admin)

运营后台不是公开后台。打开 `/admin` 后必须先输入管理员 `ADMIN_TOKEN` 登录，登录成功后 Worker 会写入短期 HttpOnly 管理会话；后台 API 仍同时校验管理员会话或 `Authorization: Bearer <ADMIN_TOKEN>`。

根路径 `https://license.lifeoskit.com` 应跳转到 `/portal`。如果线上仍返回 `Route not found`，说明 Worker 版本没有部署到包含根路径跳转的代码，需要重新部署支付 Worker。

### 3. 插件内导出授权备份

在 `Pro 授权中心` 中，用户可以导出授权备份。备份中会包含：

- 授权状态
- 到期时间
- 当前设备安装 ID
- 授权码
- 授权服务地址

建议用户在购买成功后导出一次授权备份，便于后续迁移 Vault 或更换设备。

## 四、邮件会发什么

当前邮件只用于发送验证码：

- 30 天试用验证码
- 账号中心登录验证码

当前不会主动发送：

- 支付成功通知邮件
- 授权码邮件
- 到期提醒邮件

建议后续补充 `支付成功邮件`，内容包括：

- 购买档位
- 到期时间
- 最大设备数
- 授权码
- 账号中心链接
- 换设备激活说明

推荐文案：

```text
Life OS Pro 授权已开通

你的 Life OS Pro 授权已生成。插件通常会自动激活当前设备。

授权码：LOS-XXXX-XXXX-XXXX
档位：月付 Pro
到期时间：2026-06-24
设备数：最多 3 台

你也可以随时通过账号中心找回授权：
https://license.lifeoskit.com/portal
```

## 五、换设备或重装插件

如果用户更换电脑、重装 Obsidian 或迁移 Vault：

1. 安装 Life OS 插件。
2. 打开 `Pro 授权中心`。
3. 填写授权服务地址和邮箱。
4. 在 `激活码 / 授权码` 区域粘贴授权码。
5. 点击激活当前设备。

如果用户忘记授权码，可以进入账号中心用邮箱验证码登录找回。

如果设备数达到上限，用户端应提示“设备数已满，请到账号中心查看已激活设备，或联系支持解绑旧设备/补发授权。”运营后台再处理旧设备解绑或补发新授权。

## 六、版本更新是否会导致授权失效

正常更新不会导致授权失效。

原因：

- 授权码保存在插件本地设置中。
- 授权记录保存在授权服务端。
- 插件发布包只包含 `main.js`、`manifest.json`、`styles.css`。
- 发布包不应覆盖用户的 `data.json`。

需要注意：

- 不要在模板或 release 包中附带插件 `data.json`。
- 不要随意更换授权服务的加密密钥。
- 不要删除线上 D1 数据库中的授权记录。
- 不要更改已经发出的授权码校验规则，除非提供兼容迁移。

## 七、兑换码流程

兑换码适用于早期用户、赠送用户或活动用户。

用户流程：

1. 打开 `Pro 授权中心`。
2. 在 `兑换码` 区域填写邮箱。
3. 输入兑换码。
4. 点击兑换并激活。

兑换成功后，系统会：

- 将兑换码绑定到邮箱
- 绑定当前安装 ID
- 生成授权码
- 激活当前设备
- 保存授权状态

兑换码只能使用一次。兑换后，用户仍然可以通过账号中心找回对应授权码。

## 八、后台运营使用

后台地址：

[https://license.lifeoskit.com/admin](https://license.lifeoskit.com/admin)

后台需要 `ADMIN_TOKEN` 登录。不要把后台 Token 发给普通用户，也不要把登录后的电脑借给无关人员使用。管理员退出时点击后台右上角的“退出后台”。

后台支持：

| 功能 | 用途 |
|---|---|
| 订单查询 | 按邮箱、订单号、ZPAY 交易号、支付状态查询订单 |
| 用户授权查询 | 查看用户所有授权、到期时间、设备数和设备列表 |
| 手动补发授权 | 售后、补偿、线下赠送 |
| 手动补发月付码 | 生成 30 天、3 台设备的月付 Pro 授权码 |
| 兑换码批量生成 | 生成 15 / 30 / 90 天兑换码 |
| 兑换码 CSV 导出 | 导出给运营或活动发放 |

### 售后补发建议

如果用户支付成功但插件没有自动激活：

1. 后台按邮箱或订单号查询订单。
2. 确认订单是否为 `paid`。
3. 查询该用户授权。
4. 如果授权已生成，让用户进入账号中心找回授权码。
5. 如果授权未生成，后台手动补发对应档位授权。

如果用户换设备失败：

1. 后台按邮箱查询授权。
2. 检查设备数是否已满。
3. 如确实达到上限，评估是否需要解绑旧设备或重新发放授权。

## 九、常见问题

### Q1：用户买完后为什么没有收到授权码邮件？

当前设计不是邮件发授权码，而是插件自动获取并激活。用户如果需要查看授权码，可以在账号中心用邮箱验证码登录找回。

### Q2：支付完成后没有自动激活怎么办？

先回到插件的 `Pro 授权中心`，点击继续轮询订单或等待插件自动刷新。如果仍然失败，让用户打开账号中心找回授权码，再手动粘贴激活。

### Q3：用户换电脑后授权还有效吗？

有效。用户用同一个授权码在新设备激活即可，但不能超过该档位允许的设备数。

### Q4：后续插件更新会让用户授权失效吗？

正常不会。只要更新包不覆盖用户 `data.json`，并且服务端授权数据和加密密钥不被破坏，用户授权会继续有效。

### Q5：用户看到支付通道余额不足怎么办？

这是商户侧支付通道问题，不是用户余额不足。需要开发者检查 ZPAY 商户余额或支付配置。

## 十、推荐后续优化

为了让授权体验更像正式商业产品，建议后续补充：

1. 支付成功邮件，主动发送授权信息和账号中心链接。
2. 插件 `Pro 授权中心` 增加 `我的账号` 按钮，直接打开账号中心。
3. 账号中心增加复制授权码按钮。
4. 后台增加设备解绑入口。
5. 增加授权到期提醒邮件。
6. 支付成功页提示用户回到 Obsidian 等待自动激活。
