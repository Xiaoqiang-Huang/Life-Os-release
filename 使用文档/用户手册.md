## 1. 概述

Life OS 是一个本地优先的 Obsidian 插件，帮助你在 Obsidian 中管理日记、任务、长期记忆、AI 省察和考公备考。

**核心理念：**
- **本地优先** — 数据以 Markdown 文件形式存放在你的 Obsidian 仓库中，不依赖任何云服务
- **AI 可选** — AI 回复是显式开关的，默认只记录不调用 AI
- **渐进使用** — 可以从简单的日记记录开始，逐步使用更多功能

## 2. 安装与配置

### 2.1 安装

**手动安装：**
1. 从 [GitHub Releases](https://github.com/Xiaoqiang-Huang/obsidian-life-os/releases) 下载 `main.js`、`manifest.json`、`styles.css`
2. 在你的 Obsidian 仓库中创建文件夹 `{仓库}/.obsidian/plugins/personal-life-system/`
3. 将三个文件复制到该文件夹
4. 打开 Obsidian → 设置 → 社区插件 → 启用 "Life OS"

如果你曾经把插件装在旧目录 `{仓库}/.obsidian/plugins/obsidian-life-os/`，迁移到正式目录时请同时迁移旧目录里的 `data.json`，否则授权码、安装 ID 和本地插件设置不会自动沿用。

**BRAT 安装（推荐用于自动更新）：**
1. 安装社区插件 [BRAT](https://obsidian.md/plugins?id=obsidian42-brat)
2. 命令面板执行 `Obsidian42 BRAT: Add a beta plugin for testing`
3. 输入 `https://github.com/Xiaoqiang-Huang/obsidian-life-os`
4. BRAT 会自动检查更新

### 2.2 初始配置

1. 启用插件后，点击左侧 Ribbon 栏的 sparkle ✨ 图标打开 Dashboard
2. 首次使用会提示你选择数据目录（默认 `PersonalLifeSystem/`）
3. 进入设置页（Ribbon 栏齿轮图标或 Obsidian 设置 → Life OS）：
   - **AI Provider** — 选择你的 AI 提供商（如 OpenAI、Anthropic、DeepSeek 等）
   - **Base URL** — 使用预设会自动填充，也可手动修改
   - **API Key** — 填写你的 API Key
   - **Model** — 填写模型名称（如 `gpt-4o-mini`、`claude-sonnet-4-0`）

### 2.3 基础使用路径

完成配置后，最简使用路径：

```
打开 Dashboard → 打开今日记录 → 写日记 → 打开 Chat 整理 → 查看待办 → 结束今日记录
```

## 4. Chat 面板

Chat 是与 AI 对话的面板，同时集成日记记录。

### 4.1 打开方式

- 从仪表盘点击 "Chat 对话"
- 命令面板执行 `Life OS: Open Chat`

### 4.2 模式选择

Chat 有四种模式，通过面板顶部的下拉框切换：

| 模式 | 记录位置 | AI 提示词 |
|------|----------|-----------|
| **普通聊天** | 仅 Chat 历史 | 通用对话 |
| **日记整理** | 今日日记 + Chat 历史 | 结构化输出：今日要事、一句话总结、待办延续、深度思考、AI 分析、明日计划 |
| **今日复盘** | 今日日记 + Chat 历史 | 结构化输出：完成了什么、卡点、明天最重要的一件事、待办延续 |
| **考公教练** | 今日日记 + Chat 历史 | 公务员考试辅导：判断、问题、改进动作、下一次练习 |

**记忆感知：** Chat 自动加载当前笔记、最近日记摘要、未完成待办和记忆库作为上下文。

### 4.3 AI 回复开关

- 输入框上方有一个 **「需要 AI 回复」** 复选框
- **关闭时**：点击「记录」只保存你的输入到对应位置（普通聊天仅存 Chat 历史，其他模式写入今日日记），不调用 AI
- **开启时**：点击「记录并回复」既保存输入，也调用 AI 生成回复

### 4.4 文件导入与写入位置

AI 助手可以导入 `.txt`、`.md` / `.markdown`、`.csv`、`.json` 和图片。图片识别只有在设置里启用图片视觉分析并填写视觉模型后才可用；未配置时会保留附件提示，但不会假装已经识别图片。

PDF 直接解析暂未开放。需要处理 PDF 时，建议先把内容转成文本或 Markdown，或者把关键页面截图后作为图片导入。

当 AI 识别到可保存内容时，会先让你选择写入位置：今日日记、知识库或记忆。选择后还会进入确认预览，确认后才会写入对应 Markdown 文件。

### 4.5 聊天记录管理

- **保存对话** — 手动保存当前对话到 Chat 文件夹
- **应用到今日记录** — 将 AI 回复中的结构化内容写入今日日记（仅在有 AI 回复时可用）
- 对话历史自动保存，也按设置策略定期持久化

### 4.6 使用技巧

- 日常记录建议关闭 AI 回复，快速记录想法
- 需要整理或反思时再开启 AI
- 日记整理/复盘模式下，AI 会自动生成结构化内容
- 写入日记、知识库或记忆时都要先看确认预览，不需要手动复制 AI 回复

## LLM Wiki 知识库

LLM Wiki 是 Life OS 的资料整理区。你可以在 Chat 中选择“保存到 Life OS”，也可以在 Knowledge 页面查看待整理资料；系统只处理你主动保存或明确指定的内容。

- 知识库页面会显示之前保存过的 Raw 资料、Draft 草稿、正式知识和普通知识笔记。
- 页面里的「新增资料」可以直接新建知识资料，「打开知识库目录」会生成并打开知识库索引。
- Raw 资料保存到 `Knowledge/LLMWiki/Raw`，用于存放文章、URL、当前笔记快照和其他原始资料。
- 短资料可自动整理为 Draft，Draft 保存到 `Knowledge/LLMWiki/Wiki/Drafts`。
- 长资料会先暂存，等待你选择快速整理或深度整理，不会直接写入正式 Wiki。
- Draft 可以进入 Chat 上下文，但会标记为草稿，方便你区分“可参考”和“已确认”的知识。
- 敏感资料不会自动保存、编译或进入未来上下文；需要你确认后，才会继续处理。
- 接受 Draft 前会显示发布或合并建议；只有你确认接受 Draft 后，系统才会写入正式 Wiki。
- 如果撤销 LLM Wiki 保存或整理结果，相关文件会移动到 `Knowledge/LLMWiki/Trash`。
- 插件不会自动扫描整个 Obsidian 仓库，也不会把未指定的笔记自动加入 LLM Wiki。

## 5. 任务管理

Life OS 的任务系统基于 Markdown 列表，存储在 `Tasks/` 目录。

### 5.1 任务格式

```markdown
- [ ] 完成任务 #标签 📅 2026-05-20 ^block-id
- [x] 已完成任务
```

- `- [ ]` / `- [x]` 标准 Obsidian 任务标记
- `#标签` — 可选标签，用于分类
- `📅 日期` — 可选截止日期
- `📌 日期` — 可选延续来源日期，表示任务从哪一天继承而来
- `^block-id` — 可选块引用 ID

### 5.2 操作方式

**查看待办：**
- 仪表盘自动展示前 5 条未完成待办
- 点击「未完成待办」打开 `Tasks/open.md`
- 点击「已完成待办」打开 `Tasks/done.md`

**完成任务：**
- 在仪表盘直接勾选复选框，任务自动归档到 `Tasks/done.md`
- 有 5 秒撤销时间，点击通知可撤销

**AI 提取待办：**
1. 打开一篇包含待办内容的笔记
2. 在仪表盘 AI 助手区域点击「提取待办」
3. AI 自动识别待办并追加到 `Tasks/open.md`

### 5.3 自动延续

Life OS 会在启动时和跨过午夜后检查前一天的日记：

- 前一天未完成的任务会追加到今天的日记「待办延续」区块
- 同步写入 `Tasks/open.md`，便于仪表盘统一展示
- 已存在的同名任务会自动跳过，避免重复堆积
- 延续任务会带 `#pls/carryover` 和 `📌 YYYY-MM-DD`，用于标记来源日期

如果某条任务已经完成，请在仪表盘勾选或手动改为 `- [x]`，它就不会继续被延续。

### 5.4 文件结构

```
Tasks/
├── open.md    # 所有未完成任务
└── done.md    # 已完成任务（按完成时间追加）
```

## 6. 记忆管理

记忆系统用于长期保存对你的个人信息、项目、偏好的结构化记录，AI 对话时会自动参考这些记忆。

### 6.1 记忆的产生

当你执行「结束今日记录」时，AI 会自动从当天的日记中提取重要信息，生成记忆条目并放入待确认列表。

从 0.2.0 开始，插件还会在启动时/午夜自动归档前一天记录：

- 生成 `Memory/Summaries/Daily/YYYY-MM-DD.md` 每日摘要
- 生成 `Memory/Episodes/YYYY/MM/YYYY-MM-DD.md` 事件卡
- 在原日记末尾追加「日终总结」和「延续行动」
- AI 未启用或未配置时，仍会保留基础归档说明，并延续未完成待办

### 6.2 确认记忆

1. 在仪表盘点击「记忆管理」
2. 弹窗显示所有待确认的记忆
3. 每条记忆可以：
   - **选择分类**（学业、项目、考公、人际、健康、偏好、其他）
   - **确认** — 写入对应分类文件
   - **忽略** — 标记为已忽略

### 6.3 手动添加记忆

直接编辑 `Memory/` 目录下的文件即可。

### 6.4 文件结构

```
Memory/
├── Core/
│   ├── profile.md           # 个人档案
│   └── current-projects.md  # 当前项目
├── Inbox/
│   └── pending-memories.md  # 待确认记忆
├── Summaries/
│   └── Daily/
│       └── YYYY-MM-DD.md    # 每日摘要
├── 学业.md                   # 按分类的记忆文件
├── 项目.md
├── 考公.md
├── 人际.md
├── 健康.md
├── 偏好.md
└── 其他.md
```

## 7. 报告与分析

### 7.1 日报

日报使用 AI 汇总当天的本地记录、任务、记忆和复盘上下文。当前稳定能力以本地内容分析为主；外部新闻或网页来源只有在后续明确接入并配置后才会参与。

**操作步骤：**
1. 在仪表盘点击「日报」
2. 选择报告范围或直接生成今日报告
3. AI 会结合今日记录、任务、记忆、学习打卡和复盘内容生成报告候选
4. 需要写入文件时，会先进入确认预览，确认后才保存到报告目录

**日报包含：**
- 今日日记摘要
- 今日任务、学习和复盘线索
- AI 分析总结

如果你的本地记录不足，日报会明确提示资料不足，而不是编造外部新闻或不存在的经历。

### 7.2 周报 / 月报

基于过去 N 天的日记，AI 生成跨日回顾。

- **周报** — 过去 7 天
- **月报** — 过去 30 天

### 7.3 情绪追踪

统计一段时间内日记中的情绪关键词出现频率，以文本图表方式展示。

1. 在仪表盘点击「情绪追踪」
2. 选择时间范围
3. 查看正面/负面情绪关键词统计

### 7.4 日记检索

全文搜索所有日记文件。

1. 在仪表盘点击「日记检索」
2. 输入关键词
3. 选择过滤条件（可选：情感分类）
4. 结果按相关度排序，点击可跳转到对应日记

## 8. 考公模块

考公模块需在设置中开启 `enableExamModule`。启用后在仪表盘会显示考公区域。

### 8.1 今日打卡

记录每天的学习情况。

1. 在仪表盘点击「今日打卡」
2. 填写：
   - 学习时长（分钟）
   - 完成任务数
   - 行测题数
   - 面试练习次数
   - 心情
   - 学习小结
3. 系统自动计算连续打卡天数

打卡记录存储在 `Exam/Checkins/YYYY-MM-DD.md`。

### 8.2 训练计划

查看自动生成的学习计划，按天规划学习任务。

### 8.3 行测错题本

**新增错题：**
1. 在仪表盘点击「新增行测错题」
2. 填写：
   - 题目（支持长文本）
   - 题型（言语理解/数量关系/判断推理/资料分析/常识判断）
   - 难度（简单/中等/困难）
   - 我的答案
   - 正确答案
   - 错误原因
   - 知识点
3. 确认后存入 `Exam/Xingce/` 目录

**行测统计：**
- 在仪表盘点击「行测统计」
- 按题型展示正确率、答题数量、难度分布

### 8.4 面试练习

**新增面试练习：**
1. 在仪表盘点击「面试练习」
2. 选择面试类型（综合/人际/应急/计划/社会/其他）
3. 填写题目和你的回答
4. AI 自动从 8 个维度评分：
   - 综合分析能力
   - 言语表达能力
   - 应变能力
   - 计划组织能力
   - 人际交往能力
   - 求职动机与岗位匹配
   - 举止仪表
   - 情绪稳定性

**面试趋势：**
- 在仪表盘点击「面试趋势」
- 展示各维度分数变化趋势
- 显示最频繁出现的薄弱点

### 8.5 学习目标

管理长期学习目标。

1. 在仪表盘点击「学习目标」
2. 可操作：新建、编辑、完成、弃用
3. 每个目标包含：
   - 标题
   - 目标类型（行测/面试/综合）
   - 目标值 + 单位（如 "行测达到 70 分"）
   - 起止日期
   - 优先级
   - 状态（进行中/已完成/已放弃）
   - 当前进度

目标存储在 `Exam/Goals/` 目录，YAML frontmatter 格式。

### 8.6 今日任务（学习任务）

管理每日学习任务清单。

1. 在仪表盘点击「今日任务」
2. 查看当天的学习任务
3. 可勾选完成、新增任务

任务存储在 `Exam/Tasks/YYYY-MM-DD.md`。

### 8.7 学习资料

将外部学习资料复制到知识库中，方便统一管理。

1. 在仪表盘点击「学习资料」
2. 上传文件到 `Exam/Materials/` 目录
3. 可以标记资料类型和添加说明

## 9. 主题切换

### 9.1 切换主题

在设置页可以通过下拉框或可收起的主题色块切换主题。目前下拉框和色块入口都覆盖 31 个主题，包含：

## 12. Pro 授权、购买与账号中心

Life OS 提供免费版、30 天试用、月付 Pro 和买断 Pro。授权只影响 Pro AI 能力、多设备授权和长期使用权益，不会锁住你的本地 Markdown 数据。

### 12.1 如何购买 Pro

1. 打开 Life OS 侧边栏。
2. 点击 `Pro 授权`。
3. 在授权服务设置中确认地址为 `https://license.lifeoskit.com`。

6. 点击支付宝支付按钮。

8. 回到 Obsidian，等待插件自动激活。

如果浏览器阻止自动打开支付页面，可以在 Pro 授权中心里点击 `打开网页支付`。手机端 Obsidian 如果没有拉起外部浏览器，可以复制支付链接，再粘贴到手机浏览器中打开。若订单只显示二维码，同一台手机可能无法扫码自己的屏幕，建议用另一台设备扫码，或稍后通过账号中心核对授权。

### 12.3 授权码如何获得

正常情况下，你不需要手动复制授权码。支付成功后，插件会自动查询订单、获取授权码，并激活当前设备。

授权码可以通过两种方式查看或备份：

- 在 `Pro 授权` 中导出授权备份。
- 打开 [https://license.lifeoskit.com/portal](https://license.lifeoskit.com/portal)，用购买邮箱接收验证码登录账号中心查看。

账号中心可以查看授权码、授权版本、到期时间、可激活设备数和已激活设备列表。

直接打开 `https://license.lifeoskit.com` 会进入账号中心；如果浏览器里显示接口 JSON，请改用 `https://license.lifeoskit.com/portal`。

### 12.4 换设备或重装插件

1. 在新设备安装 Life OS 插件。
2. 打开 `Pro 授权`。
3. 填写授权服务地址和邮箱。
4. 从账号中心或授权备份中找到授权码。
5. 粘贴到 `激活码 / 授权码` 区域。
6. 点击激活当前设备。

如果提示设备数已满，先在账号中心查看已激活设备，再联系开发者处理旧设备解绑或授权恢复。插件更新不会自动消耗新设备名额，除非安装 ID 或授权数据被手动清空。

### 12.5 常见购买问题

**买完没有收到授权码邮件怎么办？**  
当前邮件只用于验证码登录和账号找回。购买后的主流程是插件自动获取授权码并激活。需要查看授权码时，请进入账号中心。

**支付后没有自动激活怎么办？**  
先回到 `Pro 授权` 页面等待自动轮询，或点击继续轮询订单。如果仍未激活，可以进入账号中心找回授权码，再手动粘贴激活。

**手机端支付页面打不开怎么办？**
优先点击 `打开网页支付`，失败后复制支付链接到手机浏览器打开。如果只有二维码，换另一台设备扫码，或通过账号中心核对订单和授权。

**看到支付通道暂时不可用怎么办？**  
这通常是商户支付通道没有成功创建订单，不是你的支付宝余额不足。请稍后再试，或联系开发者处理。

**插件更新会让授权失效吗？**  
正常不会。授权记录保存在授权服务端，插件本地也保存授权状态。只要不手动删除插件数据、不超过设备数限制，更新插件不会导致授权失效。

更多细节见：[用户购买指南](用户购买指南.md)。

## 13. 隐私说明

- 插件不内置任何 API Key
- 你需要自行从 AI 提供商获取 API Key
- AI 功能会把选定的笔记内容发送到你配置的模型服务
- 报告和 AI 回复默认基于你选择的本地内容；如未来启用外部网页或新闻来源，应在对应设置中明确开启
- 插件不会把你的本地 Markdown 内容上传到插件作者服务器
- 使用 Pro 授权、购买、兑换或账号中心时，会向授权服务提交邮箱、订单信息、安装 ID 和必要的设备元数据，用于生成授权、激活设备和找回授权码
- 所有数据默认保存在你自己的 Obsidian 仓库中
- API Key 暂存在 Obsidian 插件本地数据中
- 无遥测、无分析、无作者托管的数据收集服务
