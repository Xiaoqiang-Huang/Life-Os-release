# Life OS 产品说明书

版本：0.2.0  
适用平台：Obsidian 1.5.0 及以上，桌面端与移动端均可运行  
说明范围：本文基于当前仓库源码、插件清单、已注册命令、已注册视图、设置项、服务层与授权后端代码核对，反映“当前这个插件”已经具备或已经暴露的产品能力。

## 1. 产品定位

Life OS 是一个本地优先的 Obsidian 个人生活系统插件。它把日记、任务、长期记忆、知识库、复盘、学习打卡、备考训练和 AI 助手整合到同一个 Vault 中，让用户可以用 Markdown 保存个人数据，同时用 AI 做整理、提取、总结和建议。

产品核心不是单独的聊天窗口，而是“先记录，再整理，再复盘，再沉淀”的个人系统：

- 所有核心数据优先保存在用户自己的 Obsidian Vault 中。
- AI 只在用户配置模型并主动调用时工作。
- AI 写入日记、任务、报告、记忆前会进入预览确认。
- 任务、日记、记忆、知识和复盘都可以手动编辑。
- Pro 授权与本地数据访问分离，授权状态不会锁死用户的 Markdown 数据。

## 2. 目标用户

Life OS 适合以下用户：

- 用 Obsidian 做长期个人记录、复盘和知识管理的人。
- 希望把零散想法快速沉淀成日记、任务、记忆和知识条目的人。
- 需要 AI 读取本地上下文，但又希望保留写入确认权的人。
- 正在备考、学习或训练，需要打卡、目标、错题、面试练习和趋势分析的人。
- 想把一个 Vault 逐渐建设成个人长期系统的人。

## 3. 核心价值

### 3.1 本地优先

插件自动创建一个 Life OS 数据目录，默认是 `PersonalLifeSystem/`。日记、任务、聊天、知识、记忆、复盘、报告、备考资料和授权备份都以 Markdown 或 YAML frontmatter 的形式保存在 Vault 中。

### 3.2 AI 是处理入口，不是黑箱

用户可以把想法、任务、日记片段、资料、错题、复盘问题或网页链接交给 AI 助手。AI 会结合本地上下文回答，但需要落地写入的内容必须先经过预览确认。

### 3.3 任务和记忆可控沉淀

待办可以手动创建、从日记提取、从快速记录生成，并在完成后归档。长期记忆不会直接写入正式库，而是先进入候选池，由用户确认、编辑、分类或忽略。

### 3.4 复盘形成成长轨迹

系统会把日记、任务、学习打卡和复盘摘要汇总为成长看板，展示记录天数、连续记录、完成任务、复盘次数和贡献热力图。

### 3.5 学习与备考场景内置

备考模块支持考公、考研、法考、教资和自定义备考类型。它提供学习打卡、目标、今日任务、错题统计、面试练习、面试趋势和训练计划。

## 5. 插件内主要功能

### 5.1 今日行动 Dashboard

Dashboard 是 Life OS 的工作台。它显示：

- 今日待处理任务数量。
- 今日日记是否已创建。
- 当前备考 Profile 的今日打卡状态。
- 本周复盘数量。
- 今日推荐行动。
- 今日任务前 5 条，可直接勾选完成。
- 今日日记、学习打卡、今日复盘状态卡片。
- AI 助手建议输入框。
- 快速记录、新建任务、记忆审核、打开或创建今日日记等入口。

Dashboard 会监听 Vault 文件的创建、修改和删除事件，并自动刷新状态。

### 5.2 AI 助手

AI 助手是当前产品的核心入口之一。当前源码中的 Chat 视图支持：

- 两种 UI 模式：日常对话、备考辅导。
- 三种回复风格：温和、简洁、严格。
- 三种回复长度：简短、标准、详细。
- AI 回复开关：关闭后只保存用户记录，不调用模型。
- 记入日记开关：开启后，AI 内容会先进入写回预览，确认后才写入今日日记。
- 流式生成。
- 停止生成。
- 90 秒超时保护。
- 聊天历史查看、恢复、删除和清空。
- 上下文来源抽屉，展示 AI 读取了哪些本地来源。
- 在 Chat 内切换已配置的 AI 供应商。
- 读取用户输入中的 URL，并把链接正文作为上下文之一。

AI 上下文来源包括：

- 当前打开的 Markdown 笔记及其 Obsidian 链接上下文。
- 今日日记和最近记录。
- 未完成任务和最近完成任务。
- 核心记忆、分类长期记忆和候选记忆。
- 今日复盘、最近 Daily Summary、Weekly Summary、Monthly Summary。
- 今日学习打卡和进行中的学习目标。
- 最近修改的知识库内容。
- 用户消息里的网页链接内容。

当前内置 AI Skill 只保留 `Life OS 总管`，强调本地优先、综合调度和安全写回。源码明确移除了名人、在世人物、关系 Persona 和受版权保护角色模拟。

### 5.3 AI 供应商与模型配置

设置页支持以下 AI 供应商或兼容接口：

- OpenAI 官方。
- OpenAI Compatible / 兼容代理。
- Anthropic Compatible。
- DeepSeek。
- Qwen / 百炼。
- Kimi / Moonshot。
- Hunyuan / 腾讯。
- Doubao / 火山。
- GLM / Z.AI。
- Ollama 本地模型。

配置项包括：

- API Key。
- Base URL。
- Model。
- Endpoint Path。
- Auth Header。
- Auth Prefix。
- 附加请求头 JSON。
- Anthropic API version。

设置页提供“测试连接”，会尝试列出模型并自动选择可用模型。AI 请求支持 OpenAI 风格和 Anthropic 风格接口，并在模型不支持 temperature 时尝试去掉 temperature 重试。

### 5.4 日记

日记模块支持：

- 创建或打开今日日记。
- 使用轻量日记模板。
- 使用完整版日记模板。
- 可选对接 Obsidian Daily Notes 插件目录。
- 展示今日记录状态、快速记录条数、关键词和最近内容。
- 查看最近 7 天日记。
- 查看最近快速记录。
- 一键生成今日复盘。
- 打开底层 Markdown 文件。

默认轻量模板包含：

- 今日要事。
- 一句话总结。
- 能量状态。
- 待办延续。
- 标签。
- 深度思考。
- AI 分析。
- 明日计划。

完整版模板额外包含四圣谏言区域。

### 5.5 快速记录

快速记录弹窗支持四种保存目标：

- 今日日记：追加到今日日记的快速记录区。
- 收集箱：写入 `Inbox/YYYY-MM-DD.md`。
- 待办：追加到 `Tasks/open.md`，自动生成 `#pls/task` 标签和 block id。
- 记忆候选：写入 `Memory/Inbox/pending-memories.md`，后续由用户确认。

快速记录可以从 Ribbon、命令面板、Dashboard、日记页、记忆页等入口打开，也支持把编辑器选中文本直接带入快速记录。

### 5.6 任务管理

任务模块是一个基于 Markdown 的任务看板。功能包括：

- 新建任务。
- 填写分类、截止日期、优先级、来源和备注。
- 查看今日任务、待完成、已完成和自动延续列。
- 勾选完成任务。
- 完成后归档到 `Tasks/done.md`。
- 5 秒内可撤销完成状态。
- 从今日日记提取待办。
- 手动把未完成任务延续到明天。
- 启动和跨午夜时自动检查前一天未完成任务，并延续到今天。
- 去重，避免重复任务堆积。

当 AI 未配置时，插件会使用规则提取待办。规则会识别包含“待办、明天、需要、要做、计划、完成、整理、复盘、学习、阅读、练习”等关键词的行。

### 5.7 AI 写回预览

涉及 AI 生成并写入文件的功能会进入写回预览，包括：

- 总结当前笔记。
- 从当前笔记提取待办。
- 结束今日日记并分析。
- 写入日终总结。
- 写入待办。
- 写入候选记忆。
- 写入 Daily Summary。
- 写入事件卡。
- 写入四圣谏言。
- 写入日报、周报、月报。
- Chat 内容写回今日日记。

用户可以在预览中勾选、取消或编辑写入内容，确认后才保存。

### 5.8 记忆管理

记忆系统分为候选记忆和正式分类记忆。

候选记忆支持：

- 查看所有待确认记忆。
- 编辑候选内容。
- 单条确认。
- 单条忽略。
- 批量全选。
- 批量确认。
- 批量忽略。
- 选择保存分类。
- 标记重要。
- 打开来源文件。

正式分类包括：

- 学业。
- 项目。
- 备考。
- 人际。
- 健康。
- 偏好。
- 其他。

忽略的候选记忆不会直接删除，而是进入已忽略列表，方便以后回看。

### 5.9 知识库

知识库模块用于保存可复用资料，当前有三个入口：

- 学习资料。
- 读书笔记。
- 错题知识点。

功能包括：

- 创建知识笔记。
- 打开知识库索引。
- 打开分类入口。
- 展示最近创建或修改的知识内容。
- 展示知识片段预览。

知识库数据保存在 `Knowledge/` 目录中。

- 打开日记。
- 打开打卡。

### 5.12 报告与分析

当前报告命令包括：

- 生成日报。
- 生成周报。
- 生成月报。
- 情绪追踪。
- 日记检索。

日报会基于今日记录和当天聊天记录生成总结，并写入 `Reports/daily-YYYY-MM-DD.md`。写入前需要预览确认。

周报和月报会读取最近日记，使用 AI 生成跨日回顾。若同名报告已存在，会自动生成版本文件。

情绪追踪会收集学习打卡中的心情字段，并从最近日记中识别情绪记录。若近期日记没有显式情绪字段，会尝试让 AI 推断情绪，再生成情绪分布、最近记录和 AI 分析。

日记检索支持：

- 关键词搜索。
- AI 智能问答。
- 中文时间表达解析，如今天、昨天、前天、本周、上周、本月、最近、具体日期和年月。
- 查询类型识别，如未完成事项、个人画像、话题变化、统计问题、时间查询、关键词搜索。

### 5.14 使用手册视图

插件内置使用手册视图，强调“把任何内容丢给 AI 助手”的主工作流：

1. 把原始内容发给 AI 助手。
2. 让 AI 判断要怎么处理。
3. 确认写入任务、日记、知识或记忆。
4. 回到今日行动查看结果。

手册也提供学习/备考工作流和长期使用工作流。

## 6. Pro 授权与商业化能力

### 6.1 版本对比

插件内置版本对比视图，展示：

- 免费版。
- 30 天试用。
- 月付 Pro。
- 买断 Pro。

当前产品文案定位为：

- 免费版：基础手动使用，1 台本地使用，保留本地 Markdown 数据查看、编辑、导出和迁移。
- 30 天试用：完整体验 Pro，设备数最多 3 台。
- 月付 Pro：9.9 元 / 30 天，设备数最多 3 台。
- 买断 Pro：49 元一次买断，设备数最多 5 台。

### 6.2 授权中心

Pro 授权中心支持：

- 查看当前授权状态。
- 查看到期时间。
- 查看设备数。
- 查看当前设备安装 ID。
- 保存授权服务地址。
- 保存邮箱。
- 购买月付 Pro。
- 购买买断 Pro。
- 支付宝支付。
- 创建订单。
- 展示支付二维码或支付链接。
- 轮询订单。
- 支付成功后自动激活。
- 输入兑换码并激活。
- 输入授权码并激活当前设备。
- 复制安装 ID。
- 导出授权备份。
- 打开数据说明。
- 生成迁移说明。

### 6.3 授权边界

当前授权边界分为两类：

- 免费版保留本地基础能力：Dashboard、创建/打开今日日记、快速记录、基础任务查看、新建/勾选任务、查看已有记忆、查看已有复盘、查看学习/备考数据、数据查看、导出和迁移。
- 试用和 Pro 解锁 AI 与自动化能力：AI Chat、AI 日记整理、AI 今日复盘、AI 写回日记或知识库、AI 提取长期记忆、Chat 自动加载上下文、从日记提取任务、任务自动延续、多维复盘生成、情绪/趋势统计和 AI 学习/备考辅导。

产品文案应持续强调：Pro 不锁本地 Markdown 数据，付费价值主要体现在完整 AI 体验、自动化整理、多设备授权和长期使用恢复。

### 6.4 授权码交付与找回

购买后的主流程不是邮件发送授权码，而是插件自动处理：

1. 插件创建订单并打开支付宝网页支付。
2. 支付成功后，插件轮询订单。
3. 授权服务返回 `licenseKey`。
4. 插件自动调用激活接口激活当前设备。
5. 授权码保存在插件本地设置中，可导出授权备份。

用户也可以通过账号中心找回授权码：

- 用户入口：`https://license.lifeoskit.com/portal`
- 登录方式：购买邮箱 + 邮箱验证码
- 可查看内容：授权码、授权版本、到期时间、设备额度、已激活设备列表

当前邮件只用于验证码，不主动发送授权码。后续建议增加支付成功邮件，附带授权码、账号中心链接和换设备说明。

## 11. 隐私与安全说明

Life OS 的安全边界包括：

- 不内置用户 API Key。
- API Key 保存在当前 Vault 的 Obsidian 插件数据中。
- 核心数据保存在本地 Markdown。
- AI 请求只发往用户配置的模型服务。
- AI 写入必须预览确认。
- 长期记忆必须由用户确认后才进入正式分类。
- Pro 授权不会锁死本地 Markdown 数据查看、导出和迁移。

需要联网的能力包括：

- AI 模型调用。
- Chat 中读取用户输入的 URL。
- 授权中心连接授权服务。
- 支付订单和订单轮询。
- 试用、兑换码和 License 激活。
- 日记检索的 AI 智能问答和情绪 AI 分析。

## 12. 当前版本边界与修复建议

我对“当前源码暴露了哪些功能入口”有代码级确认，但以下点需要产品文档或后续开发继续修正。

### 12.1 README 中的新闻日报描述与当前报告命令不完全一致

仓库存在 `src/news-fetcher.ts`，实现了 AI Hot、Bing RSS、Bing HTML、官方 RSS、去重和格式化能力。但当前 `src/reports.ts` 没有引用 `fetchTopicNews`、`fetchAiHotItems` 或 `formatNewsItemsToMarkdown`。也就是说，旧 README 和用户手册里“日报抓取公开新闻/AI 热点”的描述不应直接作为当前可用入口承诺。

建议修复：

- 如果要保留新闻日报卖点，把 `news-fetcher.ts` 接入 `generate-daily-report`。
- 如果暂时不接入，更新 README 和用户手册，明确当前日报只基于日记和聊天记录。

### 12.2 周/月/年自动摘要函数存在，但自动生成入口被显式跳过

源码里有生成上周、上月、上年摘要的函数，但 `checkAndGeneratePeriodicSummaries()` 当前只打印“跳过自动周期 AI 摘要，需要用户确认”的日志。

建议修复：

- 在复盘页提供显式“生成本周/本月/本年总结”按钮，并走写回预览。
- 或者删除旧自动生成函数，避免维护者误判。

### 12.3 Pro 授权边界需要持续回归

当前已经建立免费、本地基础能力与 Pro AI 能力的边界，但后续新增 AI 入口时仍需要持续回归，避免新功能绕过授权守卫。

建议修复：

- 新增 AI Chat、AI 写回、AI 复盘、AI 任务提取、AI 备考辅导等入口时，必须接入统一 entitlement guard。
- 本地数据查看、导出和迁移不得接入 Pro 锁。
- 版本对比、用户手册和购买指南要同步更新，避免用户误解“数据被锁”。

### 12.4 旧实现文件与新视图并存

仓库里同时存在旧式 `src/dashboard-view.ts`、`src/chat-view.ts`、`src/task-view.ts`、`src/quick-capture.ts` 和新式 `src/views/*`、`src/modals/*`。当前 `src/main.ts` 注册的是新视图。

建议修复：

- 明确标注旧文件为 legacy。
- 或在确认无引用后删除旧实现，减少维护歧义。

### 12.5 Release 包可能与源码存在时间差

当前仓库有未提交修改，`release/` 目录、`main.js` 和 `styles.css` 可能未必与 TypeScript 源码完全同步。

建议修复：

- 发布前运行 `npm run build`。
- 对比 release 包中的 `manifest.json`、`main.js`、`styles.css` 是否与当前源码对应。
- 更新版本号和发布说明。
