## 1. 这套模板能做什么

Life OS 是一个本地优先的 Obsidian 个人人生系统模板。它把日记、任务、AI 对话、长期记忆、复盘报告、考公备考放在同一个 Markdown 仓库里。

你可以把它当成四个系统的组合：

| 系统 | 解决的问题 | 主要入口 |
| --- | --- | --- |
| 日记与复盘 | 每天记录、收尾、总结、延续待办 | 仪表盘、今日记录、Chat |
| 任务与记忆 | 把零散想法沉淀成待办和长期记忆 | 快速记录、任务、记忆管理 |
| AI 助手 | 总结笔记、整理日记、提取待办、生成报告 | Chat、AI 助手、报告 |
| 考公备考 | 行测错题、面试练习、打卡、目标和资料 | 仪表盘的考公区域 |

核心原则：

- 数据保存在你的 Obsidian 仓库中，主要是 Markdown 文件。
- AI 是可选能力。没有 API Key 时，日记、任务、资料、打卡等功能仍可使用。
- 模板不内置任何 API Key，也不会替你注册任何模型服务。
- 你可以直接编辑所有 Markdown 文件，不需要数据库。

## 2. 你拿到压缩包后怎么打开

### 2.1 如果你拿到的是完整模板压缩包

完整模板压缩包通常叫：

```text
personal-life-system-vault-template-0.2.0.zip
```

使用步骤：

1. 解压压缩包。
2. 打开 Obsidian。
3. 点击“打开本地仓库”。
4. 选择解压后的整个文件夹，不要只选择里面的 `PersonalLifeSystem` 子目录。
5. 如果 Obsidian 提示是否信任此仓库，请选择信任。
6. 进入“设置 -> 社区插件”，确认“安全模式”已关闭，并启用“个人人生系统”插件。
7. 回到 Obsidian 主界面，点击左侧工具栏的 Life OS 图标，或打开命令面板搜索“打开 Dashboard”。

打开后建议先读三个文件：

| 文件 | 用途 |
| --- | --- |
| `README.md` | 最短上手说明 |
| `用户手册.md` | 你正在阅读的完整手册 |
| `PersonalLifeSystem/README.md` | 数据目录说明 |

### 2.2 如果你拿到的是插件压缩包

插件压缩包一般只包含三个文件：

```text
main.js
manifest.json
styles.css
```

如果你已经有自己的 Obsidian 仓库，按下面步骤安装：

1. 打开你的 Obsidian 仓库所在文件夹。
2. 进入 `.obsidian/plugins/`。
3. 新建文件夹：

```text
personal-life-system
```

4. 把 `main.js`、`manifest.json`、`styles.css` 放进这个文件夹。
5. 回到 Obsidian，进入“设置 -> 社区插件”。
6. 关闭安全模式，刷新插件列表。
7. 启用“个人人生系统”。

最终目录应类似：

```text
你的仓库/
└── .obsidian/
    └── plugins/
        └── personal-life-system/
            ├── main.js
            ├── manifest.json
            └── styles.css
```

### 2.3 如果 Obsidian 提示插件无法加载

先检查四件事：

1. 文件夹名是否是 `personal-life-system`。
2. 三个插件文件是否直接放在这个文件夹下，而不是多套了一层目录。
3. `manifest.json` 里是否能看到 `"id": "personal-life-system"`。
4. Obsidian 是否已经关闭安全模式，并允许社区插件运行。

## 3. 首次配置

启用插件后，建议按这个顺序做首次配置。

### 3.1 打开仪表盘

有三种方式：

- 点击左侧工具栏的 Life OS 图标。
- 使用命令面板，搜索“打开 Dashboard”。
- 使用命令面板，搜索“Life OS”或“个人人生系统”。

首次打开后，插件会自动创建基础目录。默认数据根目录是：

```text
PersonalLifeSystem
```

### 3.2 设置数据根目录

进入：

```text
设置 -> 社区插件 -> 个人人生系统
```

找到“数据根目录”。

默认值：

```text
PersonalLifeSystem
```

建议普通用户保持默认。只有在你已经有自己的目录规划时才修改，比如：

```text
人生系统
LifeOS
01-Life
```

注意：修改根目录后，插件会在新目录下读写数据。旧目录里的日记、任务、记忆不会自动搬过去，需要你手动迁移。

### 3.3 设置你的名称

基础设置里有三个名称：

| 设置项 | 说明 |
| --- | --- |
| 系统名称 | 仪表盘顶部显示的名字，默认“个人人生系统” |
| 助手名称 | Chat 中 AI 助手的名字，默认“小星” |
| 用户称呼 | AI 回复时如何称呼你，可以留空 |

这些设置只影响显示和 AI 语气，不影响数据结构。

### 3.4 是否兼容 Obsidian Daily Notes

设置项：

```text
兼容 Obsidian Daily Notes
```

关闭时，今日记录默认保存在：

```text
PersonalLifeSystem/Daily/YYYY-MM-DD.md
```

开启时，插件会优先使用 Obsidian Daily Notes 插件的日记目录配置。已经在用 Obsidian Daily Notes 的用户可以开启；新用户建议先保持关闭，目录更清晰。

### 3.5 是否开启考公模块

设置项：

```text
显示考公模块
```

如果你需要行测错题、面试练习、学习打卡、学习资料，就保持开启。
如果你只想用日记、任务、Chat 和记忆，可以关闭，仪表盘会更简洁。

## 4. AI 配置完整说明

Life OS 的 AI 能力包括：

- Chat 回复。
- 日记整理。
- 今日复盘。
- 总结当前笔记。
- 从当前笔记提取待办。
- 结束今日记录后的自动分析。
- 周报、月报。
- 面试练习结构化评分。
- 情绪分析和智能日记检索。
- 部分日报板块的 AI 分析。

没有配置 AI 时，上述 AI 相关功能不可用或只保留基础记录功能。

### 4.1 进入 AI 设置

路径：

```text
设置 -> 社区插件 -> 个人人生系统
```

你会看到以下主要设置：

| 设置项 | 作用 |
| --- | --- |
| AI 供应商 | 选择 OpenAI、DeepSeek、Qwen、Ollama 等 |
| API Base URL | 模型接口地址 |
| 模型名称 | 要调用的模型名 |
| API Key | 你的模型服务密钥 |
| 默认 Chat 模式 | Chat 打开时默认使用哪种模式 |
| 助手回复风格 | 控制 AI 的语气和角色 |
| 回复长度 | 简短、标准或详细 |
| Chat 写回确认 | AI 可生成日记、知识库或记忆候选，确认预览后才写入 |
| Chat 最近日记天数 | AI 参考最近多少天日记 |
| 结束今日日记后自动分析 | 日终收尾时是否调用 AI |
| 请求前自动检查模型 | 调试模型名时使用 |
| 调试模式 | 排查插件加载问题时使用 |

### 4.2 快速配置常见供应商

在“AI 供应商”里选择对应平台后，可以点击“应用当前预设”。插件会自动填入推荐的 Base URL、API 路径、请求头和默认模型。

| 供应商 | 默认 Base URL | 示例模型 |
| --- | --- | --- |
| OpenAI | `https://api.openai.com/v1` | `gpt-4.1-mini` |
| Anthropic | `https://api.anthropic.com` | `claude-3-5-sonnet-latest` |
| DeepSeek | `https://api.deepseek.com` | `deepseek-v4-flash` |
| GLM / Z.AI | `https://api.z.ai/api/paas/v4` | `glm-5.1` |
| Qwen / 百炼 | `https://dashscope.aliyuncs.com/compatible-mode/v1` | `qwen-plus` |
| Kimi / Moonshot | `https://api.moonshot.cn/v1` | `kimi-k2.5` |
| Hunyuan / 腾讯 | `https://api.hunyuan.cloud.tencent.com/v1` | `hunyuan-turbos-latest` |
| Doubao / 火山 | `https://operator.las.cn-beijing.volces.com/api/v1` | `doubao-seed-1-6-251015` |
| Ollama | `http://localhost:11434/v1` | `llama3.1`、`qwen2.5` |

模型名要以你的供应商后台实际可用的名称为准。如果调用失败，优先检查模型名和 API Key。

### 4.3 本地 Ollama 配置

如果你想尽量离线使用，可以用 Ollama。

基本步骤：

1. 安装 Ollama。
2. 拉取一个模型，例如：

```bash
ollama pull qwen2.5
```

3. 确认 Ollama 正在运行。
4. Life OS 设置中选择 `Ollama`。
5. Base URL 保持：

```text
http://localhost:11434/v1
```

6. 模型名称填写你实际拉取的模型，例如：

```text
qwen2.5
```

7. API Key 可以留空。

注意：本地模型质量、速度和上下文长度取决于你的电脑配置和模型能力。

### 4.4 OpenAI 兼容接口配置

很多服务都兼容 OpenAI 的 `/chat/completions` 格式。遇到这类服务时可以选择：

```text
OpenAI
```

或：

```text
通用兼容接口
```

常见配置：

| 项 | 示例 |
| --- | --- |
| API Base URL | `https://example.com/v1` |
| API 路径 | `chat/completions` |
| API Key 请求头 | `Authorization` |
| API Key 前缀 | `Bearer ` |

如果你的服务商给的是完整 endpoint，例如：

```text
https://example.com/v1/chat/completions
```

也可以直接填到 API Base URL。插件会尽量自动识别。

### 4.5 高级 AI 设置

高级设置适合公司代理、Azure 兼容服务或特殊网关。

| 设置项 | 说明 |
| --- | --- |
| API 路径 | 留空使用预设；支持 `{{model}}` 和 `{{apiVersion}}` |
| API Key 请求头 | 常见值：`Authorization`、`x-api-key`、`api-key` |
| API Key 前缀 | 常见值：`Bearer `；部分平台为空 |
| API 版本 | 适合 Azure 等需要版本参数的平台 |
| 附加请求头 JSON | 额外请求头，例如组织 ID、项目 ID |

附加请求头必须是合法 JSON，例如：

```json
{
  "OpenAI-Organization": "org_your_id"
}
```

### 4.6 Chat 写回设置

设置项：

```text
Chat 自动写回今日记录
```

建议新用户先关闭。关闭后，AI 回复不会立刻写入日记，你可以在 Chat 面板中预览后手动点击“写回日记”。
熟悉流程后再开启自动写回。

### 4.7 API Key 安全提醒

- 模板压缩包不包含任何 API Key。
- 你填写的 API Key 保存在 Obsidian 插件本地数据里。
- 配置过 API Key 的仓库不要直接打包发给别人。
- 需要分享模板时，请先清空 API Key，或使用全新的干净模板。
- AI 请求会把相关笔记内容发送给你选择的模型服务商，请确认你信任该服务。

## 6. 仪表盘使用方法

仪表盘是 Life OS 的首页。大多数功能都能从这里进入。

### 6.1 打开方式

- 左侧工具栏 Life OS 图标。
- 命令面板搜索“打开 Dashboard”。

### 6.2 顶部区域

顶部会显示：

- 系统名称。
- 当前日期。
- 当前主题。
- 简短提示语。

主题可以在顶部下拉菜单中切换，也可以在设置里保存。

### 6.3 今日概览

今日概览展示几个关键状态：

| 状态 | 含义 |
| --- | --- |
| 今日记录 | 今天的日记是否已创建 |
| 长期摘要 | 今天是否已生成摘要或记忆 |
| 未完成待办 | `Tasks/open.md` 中的未完成任务数量 |
| 学习打卡 | 今天是否已完成考公学习打卡 |

这些状态不是装饰信息，而是提醒你今天还有哪些闭环没完成。

### 6.4 今日工作台

常用按钮：

| 按钮 | 作用 |
| --- | --- |
| 打开今日记录 | 创建或打开今天的日记 |
| 结束今日记录 | 进行日终收尾、总结、提取待办和记忆 |
| Chat 对话 | 打开 AI Chat 面板 |
| 日历 | 打开日历视图 |

### 6.5 待办与记忆

这里会显示未完成待办的前几条。你可以：

- 直接勾选完成任务。
- 点击“展开全部”查看更多任务。
- 打开 `Tasks/open.md`。
- 打开 `Tasks/done.md`。
- 打开记忆管理弹窗。

从仪表盘勾选任务后，任务会从未完成列表移动到已完成文件，并保留完成记录。

### 6.6 AI 助手

AI 助手区域一般包含：

| 按钮 | 作用 |
| --- | --- |
| 总结当前笔记 | 对当前打开的笔记做摘要 |
| 提取待办 | 从当前笔记中提取任务，写入 `Tasks/open.md` |

如果 AI 没配置，点击后会提示你先配置 API Key。

### 6.7 报告与分析

常用入口：

| 按钮 | 作用 |
| --- | --- |
| 日报 | 生成每日资讯和个人记录摘要 |
| 周报 | 基于最近日记生成周复盘 |
| 月报 | 基于最近日记生成月复盘 |
| 情绪追踪 | 从打卡和日记中分析情绪变化 |
| 日记检索 | 搜索或智能问答历史日记 |

### 6.8 考公区域

开启考公模块后，会出现：

| 按钮 | 作用 |
| --- | --- |
| 今日打卡 | 记录学习时长、题数、心情和总结 |
| 今日训练计划 | 根据练习记录给出训练建议 |
| 新增行测错题 | 创建行测错题记录 |
| 面试练习 | 输入题目和回答，进行 AI 评分 |
| 行测统计 | 查看错题数据 |
| 面试趋势 | 查看面试练习趋势和薄弱维度 |
| 学习目标 | 管理长期学习目标 |
| 今日任务 | 管理当天学习任务 |
| 学习资料 | 添加和打开学习资料 |

## 7. 今日记录与日终分析

### 7.1 创建今日记录

打开方式：

- 仪表盘点击“打开今日记录”。
- 命令面板搜索“创建/打开今日记录”。

默认文件路径：

```text
PersonalLifeSystem/Daily/YYYY-MM-DD.md
```

### 7.2 普通模板与完整版模板

命令面板中有两个相关命令：

| 命令 | 用途 |
| --- | --- |
| 创建/打开今日记录 | 创建轻量日记 |
| 创建/打开今日记录（完整版模板） | 创建包含更多复盘区块的日记 |

建议：

- 新用户先用普通模板，降低记录压力。
- 希望每天做完整复盘时，用完整版模板。

### 7.3 日记里建议写什么

可以按这个顺序写：

```markdown
## 今日要事

- 今天发生了什么？
- 哪件事最重要？

## 情绪与状态

- 今天的精神状态如何？
- 有什么压力或开心的点？

## 待办

- 示例任务：明天继续处理的事 #pls/task 📅 2026-05-17

## 思考

- 今天有什么值得记住的判断、教训或灵感？
```

不用追求格式完美。Life OS 更适合“先记录，再整理”。

### 7.4 结束今日记录

打开方式：

- 仪表盘点击“结束今日记录”。
- 命令面板搜索“结束今日记录并分析”。

可能发生的动作：

1. 读取今日日记。
2. 调用 AI 生成总结。
3. 提取未完成待办。
4. 写入每日摘要。
5. 生成事件卡。
6. 把候选记忆写入待确认记忆。
7. 延续未完成任务。

如果 AI 未配置，插件仍会尽量做基础归档和任务延续，但不会生成高质量 AI 总结。

### 7.5 日终分析写到哪里

常见输出位置：

```text
PersonalLifeSystem/Memory/Summaries/Daily/YYYY-MM-DD.md
PersonalLifeSystem/Memory/Episodes/YYYY/MM/YYYY-MM-DD.md
PersonalLifeSystem/Memory/Inbox/pending-memories.md
PersonalLifeSystem/Tasks/open.md
```

日记本身也可能追加“日终总结”“延续行动”等内容。

## 8. 快速记录

快速记录适合在不打断当前工作的情况下捕获想法。

### 8.1 打开方式

- 左侧工具栏铅笔图标。
- 命令面板搜索“快速记录”。
- 选中文本后，命令面板搜索“快速记录选中文本”。

### 8.2 可保存到哪里

快速记录弹窗里可以选择目标：

| 目标 | 保存位置 | 适合内容 |
| --- | --- | --- |
| 今日记录 | 今日记录文件 | 当天发生的事、灵感、片段 |
| Inbox | `PersonalLifeSystem/Inbox/YYYY-MM-DD.md` | 暂时不知道放哪里的内容 |
| 未完成待办 | `Tasks/open.md` | 一句话任务 |
| 候选记忆 | `Memory/Inbox/pending-memories.md` | 值得长期记住的信息 |

### 8.3 推荐用法

- 临时想到一件事：保存到 Inbox。
- 想到明确要做的事：保存到未完成待办。
- 关于自己、项目、关系、偏好的长期信息：保存到候选记忆。
- 当天的片段记录：保存到今日记录。

## 9. 任务管理

Life OS 的任务是标准 Markdown 任务。

### 9.1 未完成任务文件

路径：

```text
PersonalLifeSystem/Tasks/open.md
```

示例：

```markdown
- 示例任务：整理面试自我介绍第一版 #pls/task 📅 2026-05-18 ^pls-task-001
```

### 9.2 已完成任务文件

路径：

```text
PersonalLifeSystem/Tasks/done.md
```

从仪表盘勾选完成后，任务会被移动或归档到这里。

### 9.3 任务字段说明

| 片段 | 含义 |
| --- | --- |
| `- [ ]` | 未完成 |
| `- [x]` | 已完成 |
| `#pls/task` | Life OS 任务标签 |
| `📅 YYYY-MM-DD` | 截止日期 |
| `📌 YYYY-MM-DD` | 从哪一天延续而来 |
| `^block-id` | Obsidian 块引用 |

### 9.4 从笔记提取待办

步骤：

1. 打开一篇笔记。
2. 打开仪表盘。
3. 点击“提取待办”。
4. AI 会识别笔记里的行动项。
5. 确认后写入 `Tasks/open.md`。

适合从会议记录、日记、复盘、学习笔记中抽取任务。

### 9.5 跨日延续

如果昨天日记或任务文件里有未完成任务，Life OS 会尝试把它们延续到今天。

延续任务通常带有：

```text
#pls/carryover
📌 YYYY-MM-DD
```

避免重复的建议：

- 完成的任务及时勾选。
- 不要在多个地方手动复制完全相同的任务。
- 每天结束时使用“结束今日记录”做一次收尾。

## 10. 长期记忆管理

长期记忆用于让 Life OS 逐渐理解你的背景、项目、偏好和重要事实。

### 10.1 记忆分几类

默认分类：

| 分类 | 示例 |
| --- | --- |
| 学业 | 专业、课程、学习计划 |
| 项目 | 正在做的产品、代码仓库、交付计划 |
| 考公 | 目标岗位、备考进度、薄弱题型 |
| 人际 | 重要联系人、沟通偏好 |
| 健康 | 作息、运动、饮食、状态 |
| 偏好 | 你喜欢的工作方式、表达风格 |
| 其他 | 暂时无法归类的信息 |

### 10.2 候选记忆

AI 或快速记录会把待确认的信息放到：

```text
PersonalLifeSystem/Memory/Inbox/pending-memories.md
```

候选记忆不会自动变成正式记忆。你需要在“记忆管理”中确认。

### 10.3 打开记忆管理

方式：

- 仪表盘点击“记忆管理”。
- 命令面板搜索“记忆管理”。

你可以对每条候选记忆执行：

| 操作 | 结果 |
| --- | --- |
| 确认 | 写入对应分类文件 |
| 忽略 | 不再作为正式记忆 |
| 修改分类 | 改到更合适的记忆分类 |

### 10.4 核心记忆文件

建议认真维护这两个文件：

```text
PersonalLifeSystem/Memory/Core/profile.md
PersonalLifeSystem/Memory/Core/current-projects.md
```

`profile.md` 适合写：

- 你的身份背景。
- 长期目标。
- 工作和学习偏好。
- 不希望 AI 忘记的重要约束。

`current-projects.md` 适合写：

- 当前正在推进的项目。
- 每个项目的目标、状态、下一步。
- 项目的关键路径和风险。

### 10.5 记忆写作建议

好的记忆应该具体、稳定、可复用。

推荐：

```markdown
- 用户正在准备公务员面试，当前重点是综合分析题和应急应变题。（2026-05-16，来源：Daily/2026-05-16.md）
```

不推荐：

```markdown
- 今天很累。
```

“今天很累”更适合放在日记或情绪记录里，不一定要进入长期记忆。

## 11. Chat 面板

Chat 是 Life OS 的 AI 对话和日记整理中心。

### 11.1 打开方式

- 仪表盘点击“Chat 对话”。
- 命令面板搜索“打开 Chat 面板”。

### 11.2 四种模式

| 模式 | 适合场景 | 主要特点 |
| --- | --- | --- |
| 普通聊天 | 问问题、讨论想法、让 AI 参考你的笔记 | 输出自由 |
| 日记整理 | 把口语化记录整理成结构化日记 | 适合写回今日记录 |
| 今日复盘 | 收尾一天，找完成、卡点、明日重点 | 更偏复盘 |
| 考公教练 | 备考、公考表达、题目讨论 | 更偏训练和纠错 |

### 11.3 “需要 AI 回复”开关

Chat 面板有一个“需要 AI 回复”开关。

关闭时：

- 只记录你的输入。
- 不调用 AI。
- 不消耗模型额度。

开启时：

- 保存你的输入。
- 调用 AI 生成回复。
- 可以继续生成日记、知识库或记忆候选，但写入前会先进入确认预览。

### 11.4 “记入 / 写回确认”开关

如果开启“记入”，AI 会在需要保存时生成候选内容。
候选内容不会直接落盘，必须经过你在预览弹窗中确认后，才会写入今日日记、知识库或记忆。

### 11.5 文件导入

输入框支持导入文本、Markdown、CSV、JSON 和图片。图片识别需要在设置里单独启用视觉分析并填写视觉模型；没有视觉模型时，插件会提示图片不可识别，不会把图片内容当成已读。

PDF 直接解析暂未开放。需要处理 PDF 时，先转成文本或 Markdown，或者把关键页面截图后按图片导入。

### 11.6 Chat 会参考哪些上下文

根据设置和当前状态，Chat 可能参考：

- 当前打开的笔记。
- 最近若干天日记。
- 未完成待办。
- 长期记忆。
- 今日记录。
- 考公练习记录。

“Chat 最近日记天数”控制它参考最近多少天日记，默认是 7 天。

### 11.6 Chat 历史保存

设置项：

```text
Chat 历史保存
```

可选：

| 选项 | 含义 |
| --- | --- |
| 不保存 | 关闭后不保留长期对话记录 |
| 仅保存摘要 | 保存简短摘要，推荐默认 |
| 完整保存 | 保存完整对话内容 |

保存位置：

```text
PersonalLifeSystem/Chat/
```

### 11.7 助手风格

可选风格：

| 风格 | 适合场景 |
| --- | --- |
| 温柔陪伴 | 日记、情绪、自我理解 |
| 高效执行 | 任务拆解、计划推进 |
| 严格教练 | 复盘、纠偏、执行监督 |
| 考公导师 | 公考学习、申论、面试 |
| 四圣 | 更偏省察和多角度建议 |
| 自定义 | 你自己写系统提示词 |

如果选择“自定义”，建议写清楚：

- AI 的角色。
- 回复结构。
- 语气边界。
- 是否先指出风险。
- 是否必须给下一步行动。

### 14.7 面试练习

打开方式：

- 仪表盘点击“面试练习”。
- 命令面板搜索“打开面试练习”。

支持的题型包括：

- 自我介绍。
- 综合分析。
- 应急应变。
- 人际关系。
- 组织协调。
- 职位认知。
- 情景模拟。
- 无领导小组讨论。
- 专业专项。

使用步骤：

1. 选择题型。
2. 输入题目。
3. 输入你的回答。
4. 点击 AI 结构化评分。
5. 查看评分、优点、待改进、建议和下次练习。
6. 满意后保存记录。

AI 评分维度包括：

| 维度 | 含义 |
| --- | --- |
| 综合分析能力 | 是否能理解问题、抓住本质、多角度分析 |
| 言语表达能力 | 表达是否清楚、准确、流畅 |
| 应变能力 | 面对突发情况能否稳妥处置 |
| 计划组织协调能力 | 能否制定方案、协调资源、复盘效果 |
| 人际交往意识与技巧 | 能否处理领导、同事、群众关系 |
| 自我情绪控制 | 压力下能否保持稳定 |
| 求职动机与拟任职位匹配性 | 动机和岗位是否匹配 |
| 举止仪表 | 仪态、语速、礼貌是否符合场景 |
| 专业能力 | 是否体现岗位相关专业理解 |

保存位置：

```text
PersonalLifeSystem/Exam/Interview/
```

### 14.8 面试趋势

## 16. 推荐工作流

### 16.1 五分钟开始法

适合第一天使用：

1. 打开仪表盘。
2. 点击“打开今日记录”。
3. 写三句话：今天做什么、现在状态如何、最重要的一件事是什么。
4. 用快速记录添加一个待办。
5. 晚上点击“结束今日记录”。

### 16.2 每日工作流

早上：

1. 打开仪表盘。
2. 查看未完成待办。
3. 打开今日记录。
4. 写下今天最重要的 1 到 3 件事。

白天：

1. 用快速记录捕获想法。
2. 明确任务直接存到未完成待办。
3. 重要事实存到候选记忆。

晚上：

1. 补充今日记录。
2. 打开 Chat，用“今日复盘”模式整理。
3. 点击“结束今日记录”。
4. 在记忆管理里确认重要记忆。

### 16.3 每周复盘

每周固定一天：

1. 生成周报。
2. 查看本周反复出现的卡点。
3. 清理 `Tasks/open.md`。
4. 更新 `Memory/Core/current-projects.md`。
5. 给下周设定 1 到 3 个重点。

### 16.4 考公备考工作流

每天：

1. 在“今日任务”里添加学习任务。
2. 做题后把错题录入“新增行测错题”。
3. 做一次面试练习，尽量保存 AI 结构化评分。
4. 晚上做“今日打卡”。

每周：

1. 查看“行测统计”。
2. 查看“面试趋势”。
3. 根据薄弱维度调整下周训练。
4. 更新学习目标。

### 16.5 不使用 AI 的工作流

如果你不想配置 AI，也可以这样用：

- 今日记录：手动写日记。
- 快速记录：捕获想法和待办。
- 任务管理：维护 `Tasks/open.md` 和 `Tasks/done.md`。
- 考公模块：记录错题、任务、目标、打卡和资料。
- 日历：查看日记和打卡分布。

AI 只是增强，不是使用门槛。

## 17. 备份、迁移与隐私

### 17.1 如何备份

Life OS 数据都在 Obsidian 仓库里。你可以用：

- Obsidian Sync。
- Git。
- OneDrive、iCloud、坚果云等同步盘。
- 定期复制整个仓库。
- 定期压缩 `PersonalLifeSystem/`。

### 17.2 迁移到另一个仓库

如果只迁移业务数据，复制：

```text
PersonalLifeSystem/
```

如果连插件和设置也一起迁移，复制：

```text
.obsidian/plugins/personal-life-system/
```

注意：插件设置里可能包含 API Key。分享给别人前不要复制包含你个人 API Key 的插件数据。

### 17.3 分享模板给别人

推荐分享干净模板，而不是你正在使用的个人仓库。

分享前检查：

1. 删除或清空个人日记。
2. 清空 API Key。
3. 检查 `Memory/` 是否有个人隐私。
4. 检查 `Chat/` 是否有对话历史。
5. 检查 `Reports/` 是否包含个人信息。

### 17.4 AI 与隐私

只有在你使用 AI 功能时，相关笔记内容才会发送给你配置的模型服务。
插件本身没有作者托管的数据收集服务，也不需要注册 Life OS 账号。

## 18. 常见问题与排查

### 18.1 左侧没有 Life OS 图标

检查：

1. 社区插件是否已启用。
2. “个人人生系统”插件是否已启用。
3. 插件目录是否正确。
4. 重启 Obsidian。

### 18.2 启用插件时报错

检查：

1. `main.js` 是否存在。
2. `manifest.json` 是否存在。
3. `styles.css` 是否存在。
4. 插件文件是否来自同一个版本。
5. Obsidian 版本是否不低于 1.5。

### 18.3 AI 总是调用失败

按顺序排查：

1. API Key 是否填写。
2. 模型名称是否真实存在。
3. Base URL 是否正确。
4. API 路径是否正确。
5. 网络是否能访问服务商。
6. 账户余额或额度是否充足。
7. 如果是公司网络，是否需要代理。
8. 打开“请求前自动检查模型”做一次检查。

### 18.4 DeepSeek、Qwen、Kimi 等国产模型怎么填

优先选择对应供应商，然后点击“应用当前预设”。
再填写 API Key，确认模型名和你后台开通的模型一致。

### 18.5 Ollama 没有回复

检查：

1. Ollama 是否正在运行。
2. 模型是否已经拉取。
3. Base URL 是否是 `http://localhost:11434/v1`。
4. 模型名是否和 `ollama list` 看到的一致。
5. 本地模型首次加载可能较慢。

### 18.6 今日记录没有创建

检查：

1. 数据根目录是否设置正确。
2. 如果开启 Daily Notes 兼容，检查 Obsidian Daily Notes 插件配置。
3. 仓库是否有写入权限。
4. 文件名是否被同步软件锁定。

### 18.7 任务重复出现

可能原因：

- 手动复制了相同任务。
- 昨天的任务没有勾选完成。
- 多个文件里写了相同的待办。

处理方法：

1. 打开 `Tasks/open.md`。
2. 删除重复任务。
3. 已完成的任务改为 `- [x]` 或在仪表盘勾选。
4. 每天结束时使用“结束今日记录”收尾。

### 18.8 行测统计为空

检查错题文件是否在：

```text
PersonalLifeSystem/Exam/Xingce/
```

并且 frontmatter 包含：

```yaml
type: xingce-question
```

### 18.9 面试趋势为空

原因通常是还没有保存带结构化评分的面试记录。
先在“面试练习”中使用 AI 结构化评分，并保存记录。

### 18.10 情绪追踪没有数据

可以通过这些方式增加数据：

- 在学习打卡里选择心情。
- 在日记中写“情绪：焦虑”或类似表达。
- 配置 AI，让插件推断最近日记情绪。

### 18.11 日报没有新闻

可能原因：

- 网络不可用。
- 搜索来源临时不可用。
- 话题过于冷门。
- AI 没有配置。

处理方法：

- 换一个更具体的话题。
- 稍后再试。
- 配置 AI 后重新生成。

### 18.12 手机端能不能用

插件声明支持移动端。实际体验取决于：

- 你的 Obsidian 移动端是否允许社区插件。
- 你的同步方式是否稳定。
- AI 服务是否能从手机网络访问。

大量弹窗和长文本输入在桌面端体验更好。

## 最后一页：新用户只要记住这 5 件事

1. 打开模板后，先启用“个人人生系统”插件。
2. 不配置 AI 也能用日记、任务、打卡、资料和日历。

5. 所有数据都在 `PersonalLifeSystem/`，可以直接编辑、备份和迁移。
